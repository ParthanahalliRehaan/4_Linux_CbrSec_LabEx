# ЁЯФР Hydra (Password Cracking Tool)
**Purpose**: Automates brute-force/dictionary attacks against login services.

### Options
- **`-l USERNAME`** тЖТ single username.  
  Example: `hydra -l admin ...`
- **`-L FILE`** тЖТ file with multiple usernames.  
  Example: `hydra -L usernames.txt ...`
- **`-p PASSWORD`** тЖТ single password.  
  Example: `hydra -p password123 ...`
- **`-P FILE`** тЖТ file with multiple passwords.  
  Example: `hydra -P passwords.txt ...`
- **`-s PORT`** тЖТ specify port.  
  Example: `hydra -s 8080 ...`
- **`http-post-form`** тЖТ attack HTTP login forms.  
  Example:  
  ```
  http-post-form "/login:username=^USER^&password=^PASS^:Invalid username or password"
  ```
- **`-o FILE`** тЖТ save results to file.  
  Example: `hydra -o results.txt ...`
- **Other protocols**: `ssh`, `ftp`, `http-get`, `smtp`, etc.  
  Example: `hydra -l root -P pass.txt ssh://192.168.1.20`

# ЁЯУЭ echo
**Purpose**: Print text or write text into files.

### Options
- **`echo "text"`** тЖТ print text.  
- **`echo -n "text"`** тЖТ no newline at end.  
- **`echo -e "line1\nline2"`** тЖТ interpret escape sequences:
  - `\n` тЖТ newline  
  - `\t` тЖТ tab  
  - `\\` тЖТ backslash  
- **Redirect output**:
  - `>` тЖТ overwrite file.  
    ```bash
    echo "hello" > file.txt
    ```
  - `>>` тЖТ append to file.  
    ```bash
    echo "world" >> file.txt
    ```

# ЁЯУВ cat
**Purpose**: Concatenate and display file contents.

### Options
- **`cat file.txt`** тЖТ show file contents.  
- **`cat file1 file2`** тЖТ show multiple files sequentially.  
- **`cat > file.txt`** тЖТ create file, type content, save with `CTRL+D`.  
- **`cat >> file.txt`** тЖТ append typed content.  
- **`cat -n file.txt`** тЖТ show line numbers.  
- **`cat -b file.txt`** тЖТ show line numbers, skip blank lines.  
- **`cat -s file.txt`** тЖТ squeeze multiple blank lines into one.  
- **`cat -E file.txt`** тЖТ show `$` at line ends.  
- **`cat -T file.txt`** тЖТ show `^I` for tabs.

---

# ЁЯУЦ head
**Purpose**: Show the beginning of files.

### Options
- **`head file.txt`** тЖТ first 10 lines (default).  
- **`head -n 5 file.txt`** тЖТ first 5 lines.  
- **`head -c 20 file.txt`** тЖТ first 20 characters.  
- **`head -q file1 file2`** тЖТ suppress file name headers.  
- **`head -v file.txt`** тЖТ always show file name header.

# ЁЯзй Examples in Practice
```bash
echo -e "user1\nuser2\nuser3" > usernames.txt
cat -n usernames.txt
head -n 2 usernames.txt
hydra -L usernames.txt -P passwords.txt localhost -s 8080 http-post-form "/:username=^USER^&password=^PASS^:Invalid username or password" -o results.txt
cat results.txt
```

# ЁЯУМ Key Security Lessons
- Hydra shows how **automation cracks weak passwords fast**.  
- **Manual attempts** are slow; Hydra demonstrates why password policies matter.  
- **Weak/common passwords** are easily broken.  
- **Real attackers use huge dictionaries** (millions of passwords).  
- Defenses: strong unique passwords, account lockouts, CAPTCHAs, MFA.
